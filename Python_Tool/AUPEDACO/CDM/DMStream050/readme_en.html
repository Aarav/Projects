1. Overview

DiskMarkStream is an automatic macro tool specially designed for CrystalDiskMark (CDM) created by hiyohiyo. It will start CDM with predefined settings and at completion of each test, will save screenshot and log then move on to next test. This consecutive unattended operation will help user to test various conditions and multiple drives at ease.

Note: This software is created by EMO having no direct relation with hiyohiyo, author of CDM. So if you have bug report, suggestion, request or any other message regarding this software, please send it to EMO.

2. Requirements

  • CrystalDiskMark 3.0.2 (including Shizuku Edition)
  • Windows 8.1 Preview, Windows 8, Windows 7 and Windows XP
  • .NET Framework 4.0

3. How to run

  1. Start this software. Press [Import macro file] and specify a macro file. Then its content will be reflected to list (You CANNOT edit the list directly).
  2. Start CDM and make its whole window visible inside the desktop. Be sure that CDM's window will not lap over this software's window.
  3. Press [Proceed]. Unless there is any problem in CDM, a message saying "Confirmed CDM is up. Start testing." will pop up. So press [Yes].
  4. Test will begin and status box of this software will turn to, through "Waiting" and "Using cursor", "Testing". DO NOT move cursor during "Using cursor". You can use cursor during "Waiting" and "Testing" but during "Testing", NEVER move CDM's window nor hide it by other window (If it happens, this software WILL NOT be able to determine when CDM finishes testing). This software is basically for unattended operation.
    If you wish to interrupt testing, press [Interrupt] (It WILL NOT stop CDM automatically). To resume testing, press [Resume]. To end testing at this stage, press [Finish].
  5. After CDM finished all tests, the status box will turn to "Finished". CDM's screenshot and log will be saved in the following files in "result" folder:
    Screenshot of each test MMddHHmmss.png
    Log of each test MMddHHmmss.txt
    Accumulated log MMdd.txt
    Accumulated log converted into csv format MMdd.csv
    (MM: month, dd: day, HH: hour, mm: minute, ss: second)

4. Making macro file

The format of macro file is same as standard csv format. So, it is easy to make it with Excel or other spreadsheet software and save it as a csv file. The order of items is same as the list in this software's window (except number in far left). The values of respective items are as follows:

If a field is left blank or there is no such setting in CDM, the default value will be used.

For example, if test type is Seq, number of test runs is 3, test size is 100MB, test drive is 2nd one, test data is Random (default) and comment is "Drive test #1", the row will be like this:

Seq 3 100 2 Drive test #1

This content is identical to the following text:

Seq, 3, 100, 2, , Drive test #1

See included sample macro file (samplemacro.csv) for reference.

5. Settings

Details of items in Settings Dialog.

  • [Queue time] : A period of time (milliseconds) that this software will wait between clicking or other actions when using cursor. The shortest is 100. I think it will be enough for most cases but if response seems not to catch up, make it longer. The longest is 1000 (1 second).
  • [Test interval] : A period of time (seconds) between tests. The shortest is 10 and the longest is 3600 (1 hour).
  • [Macro file] :
    • [Specify text encoding] : If a macro file seems not to be correctly interpreted, specify exact text encoding.
    • [Use only ASCII code in comment] : If unchecked, characters other than ASCII code will become usable in comment. However, it may take longer time to save screenshot and log. In such case, this software will wait up to 1 minute.
  • [Batch file to be executed between tests] : A batch file can be executed between tests. It will start just after the end of each test.
    • [Wait for completion of batch execution] : This software will wait for the completion of batch execution before starting "Waiting" (for next test) period. During the batch execution, status box will show "Executing Batch". If unchecked, this software will not care for the fate of batch execution except next time of execution. If previous execution will not have exited, no new execution will be kicked off.
    • [Execute as administrator] : This software will execute the batch file as administrator, provided that this software itself is executed by administrator (To avoid being required for confirmation each time of batch execution). If executed by administrator, [Enabled] will be shown and if not, [Disabled] will be.
  • [Log file to be created] :
    • [Use comment for file names] : Comment will be used to name screenshot and log files. Be aware that if the same comment, the files will be overwritten.
  • [Audio file to be played when all tests finished] : This software can play back audio file to nofity that all tests finished. Playable audio files are the same as those of Windows Media Player such as MP3, WMA, WAV, MID. If you don't want it, just leave it blank.

6. Other

This software will get partial image (30x30 pixels) of CDM's button and calculate its hash code every 2 seconds during testing. The code will be compared with that saved before testing so as to determine whether CDM finishes all tests. No file on storage will be red or written during testing. Therefore, I do not think it will make any significant difference in CDM's results. However, it might change case by case.

Regarding multi-language, storing a language file in "language" folder will let this software to detect the file at its startup and make it selectable. As for the format of language file, refer to English version "English.lang". The text encoding is UTF-8 and the maximum length of expression of each item is 128 letters.
